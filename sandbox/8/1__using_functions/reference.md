## Дополнительные материалы

**1.** Есть способ задать в программе что-то вроде реально неизменяемого константного значения. Для этого используется директива препроцессора `#define`. Мы изучим, как она работает в уроке, посвящённом массивам.

**2.** Насколько серьёзно квалификатор типа `const` защищает переменную? Ну, как сказать... Относительно хорошо. Случайно, конечно, такую переменную изменить не получится, но если очень захотеть, то, способ имеется. Для этого нужно использовать указатели. Наше первое знакомство с ними состоится буквально в следующей заметке!

**3.** Интересный факт. Функция, при необходимости, может вызвать сама себя. Такая ситуация называется =рекурсией=, а вызов называется =рекурсивным вызовом=. Использование рекурсии в некоторых случаях позволяет упростить и сделать более наглядным решение задачи. В текущей версии Курса эта тема разбираться не будет. Если вас эта тема заинтересовала, то рекомендую к просмотру [видео из курса по языку Си от Т. Хирьянова](https://cs.mipt.ru/c_intro/lessons/lesson4.html).

**4.** В уроке про инструкции управления циклом я вам обещал показать ещё один вариант, как можно выйти сразу из нескольких вложенных циклов. Там я описал вариант, когда мы используем дополнительную переменную-флаг. Получаем что-то вроде такого кода:

```c
// код до циклов

int break_flag = 0;

for (int i = 0; i < N; i++) {
    for (int j = 0; j < M; j++) {
        for (int k = 0; k < K; k++) {
            
            if (условие завершения всех трёх циклов) {
                break_flag = 1; // ставим значение флага 1
                break;          // выходим только из цикла по k
            }
        }
        if (break_flag) break;  // если флаг 1, то выходим из цикла по j
        // какой-то код
    }

    if (break_flag) break;      //  если флаг 1, то выходим из цикла по i
    // ещё какой-то код
}

// код после циклов
```

Другой вариант -- вынести блок с циклами в отдельную функцию и вызвать её. Код выше превратится во что-то такое:

```c
// код до циклов

nested_cycles(N, M, K);

// код после циклов

... 

// объявление функции с вложенными циклами
void nested_cycles(int N1, int N2, int N3)
{
    for (int i = 0; i < N1; i++) {
        for (int j = 0; j < N2; j++) {
            for (int k = 0; k < N3; k++) {
                
                if (условие завершения всех трёх циклов) {
                    return; // завершаем выполнение функции 
                            // и автоматически выходим из всех вложенных циклов
                }

            }
            // какой-то код

        }
        // ещё какой-то код

    }
}
```


При этом, мы могли бы даже вернуть какое-то значение из циклов, если бы использовали не void-функцию, а функцию, которая возвращает значение.

Но чтобы не городить подобный огород, в очередной раз повторяю, избегайте чрезмерной вложенности циклов!
