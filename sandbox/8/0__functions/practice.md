- =Неопределённое поведение= -- это ситуация, когда стандарт языка **не описывает**, что должно произойти в таком случае. Программа, содержащая неопределённое поведение может "случайно" работать правильно, может работать неправильно, может вообще упасть и вести себя различно при разных запусках и на разных компиляторах. При этом компилятор **не обязан** даже выдавать ошибку -- иногда он лишь предупреждает, иногда молчит. Замените реализацию функции `max2` в Листинге 5 на ту, что предложена ниже. В ней допущена сознательная ошибка: не обрабатываются ситуации, когда значение `a` меньше `b`. Поэтому, если передать в эту функцию, например, `3 7`, то мы получим неопределённое поведение, т.к. функция должна вернуть значение типа `int`, а в этом случае она ничего не возвращает. Проверьте, как обработает эту ситуацию ваша IDE. О результатах напишите в комментариях к уроку.
```c
int max2(int a, int b)
{
        // две точки выхода из функции
        if (a > b) {
                return a;
        } else {
                printf("UB!!!\n");
                // "случайно" забудем return в этой ветке
                // return b; 
        }
}
```

- Как и сказано в уроке, стандарт языка Си не позволяет объявлять функции внутри других функций. Но при этом разработчики компиляторов иногда расширяют возможности стандартов, поэтому у некоторых код из Листинга 6 нормально скомпилируется и отработает. Попробуйте скомпилировать и запустить Листинг 6 у себя в системе. Получилось ли у вас? Напишите об этом в комментариях к уроку с указанием вашей IDE и используемого компилятора.
