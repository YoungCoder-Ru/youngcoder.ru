# Указатели в языке Си. Первое знакомство

Итак, пришло время познакомиться поближе с указателями! 

Возможно вы наслышаны, что это очень сложная и запутанная тема, но, я вас уверяю, не стоит переживать. Мы затронем только самые основы, ничего грандиозно сложного не будет.

## Переменные и адреса в памяти компьютера

Как нам уже известно из урока про функцию `scanf`, у каждой переменной в нашей программе есть адрес в памяти. Чтобы его получить, мы используем оператор взятия адреса `&`. Давайте на них посмотрим. Создадим три переменные разных типов и выведем их адреса на экран, воспользовавшись функцией `printf` и спецификатором `%p`.

Листинг 1: Адреса переменных в памяти
```c
# include <stdio.h>

int main(void)
{
        // создаём и инициализируем переменные различных типов
        char symbol = 'A';
        int num = 5;
        double real = 3.1415926;
        
        // выводим адреса переменных на экран
        printf("Variable symbol address: %p\n\n", &symbol);
        printf("Variable num address: %p\n\n", &num);
        printf("Variable real address: %p\n\n", &real);

        return 0;
}
```

![Рис.1 Адреса переменных в оперативной памяти](./variable_addresses.png "Рис.1 Адреса переменных в оперативной памяти")

Эти адреса будут меняться при каждом запуске программы. 

Чтобы ещё лучше зафиксировать связь между переменными и адресами, представим память, выделенную программе, в виде длинной ленты, состоящей из одинаковых пронумерованных ячеек. Каждая ячейка -- это один байт (`8` бит). Так вот, номер ячейки -- это и есть адрес переменной в памяти. Спецификатор `%p` выводит адрес ячейки в шестнадцатеричном виде, а не в десятичном.

На рисунке ниже я схематично изобразил кусок оперативной памяти и переменные в ней.

![Рис.2 Размещение переменных в оперативной памяти](./ram_snapshot.png "Рис.2 Размещение переменных в оперативной памяти")

Количество ячеек, которое занимает переменная в памяти, определяется типом данных. У меня в системе переменная `symbol` занимает одну ячейку, `num` -- четыре ячейки, а `real` -- восемь ячеек. У вас переменная типа `char` тоже будет занимать `1` байт, а вот размеры переменных других типов уже могут отличаться. Во втором уроке была даже отдельная исследовательская задача для хакеров, где предлагалось выяснить эти размеры для вашей системы. 

% **Обратите внимание:** для переменных, которые занимают несколько ячеек, в качестве адреса возвращается номер первой из них (с минимальным значением).

Теперь, когда с адресами разобрались, можно переходить к основной теме урока -- указателям.

## Что такое указатель?

Начнём с того, что попытаем "поженить" два уже известных нам факта о языке Си.
1. Все функции в языке Си работают с копиями значений, которые им передаются через аргументы. Для этого при вызове функции создаются локальные переменные, описанные в заголовке функции.
2. В функцию `scanf` передаются адреса переменных, а не сами переменные. 

Допустим нам требуется считать с клавиатуры число в переменную `num`. Для этого мы пишем: `scanf("%d", &num);`. Здесь в качестве второго аргумента мы передаём в функцию `scanf` адрес переменной `num`. А значит в этот момент где-то должна быть создана локальная переменная в которую записывается копия переданного адреса. 

Мы уже знаем, как работать с символами, целыми и вещественными числами, но тут совсем другое дело. Здесь нужно создать переменную для хранения адреса. Мы это делать не умеем, но раз функция `scanf` работает, то стало быть в языке Си существуют какие-то механизмы для работы с адресами. Этот абсолютно правильный вывод подводит нас к указателям.

=Указатель (pointer)= -- это переменная, которая используется для хранения адреса.

И это всё? Ну, не совсем. Ведь мало знать, что указатели существуют, надо же ещё и научиться ими пользоваться. Именно этим и предлагаю заняться.

## Объявление указателя в C

Сперва нам нужно научиться создавать указатели. Т.к. указатель это переменная, то и объявляется он по тем же самым правилам: сначала указываем тип данных, а затем имя переменной.

Листинг 2. Объявление указателей
```c
int * p_num;        // объявляем переменную p_num типа "int *"
                    // p_num -- имя переменной-указателя
                    // int * -- тип данных (указатель на int или адрес int)

char * p_symbol;    // объявляем переменную p_symbol типа "char *"
                    // p_symbol -- имя переменной-указателя
                    // char * -- тип данных (указатель на char или адрес char)
```

В Листинге 2 мы создали два указателя: `p_num` и `p_symbol`. Теперь в переменную `p_num` мы можем сохранить адрес любой переменной типа `int`. А в указатель `p_symbol` -- адрес любой переменной типа `char`.

Для каждого из известнымх нам базовых типов данных имеется соответствующий ему =адресный (указательный) тип данных=:
- `char *` -- указатель на `char` или адрес `char`
- `int *` -- указатель на `int` или адрес `int`
- `double *` -- указатель на `double` или адрес `double`
- `float *` -- указатель на `float` или адрес `float`

Имя указателя подчиняется стандартным ограничениям, накладываемым на имена переменных: только латиница, цифры и знак подчёркивания. Для наглядности, чтобы сразу отличать указатели от любых других переменных, я буду добавлять к именам указателей префикс `p_` (сокращение от английского слова pointer).

## Сохранение адреса в указателе

Для того, чтобы присвоить указателю какое-то значение (сохранить адрес в указателе) используется всё тот же оператор присваивания.

Давайте рассмотрим какой-нибудь простой пример.

Листинг 3. 
```c
#include <stdio.h>

int main(void)
{
        int age = 0;
        int *p_age; // объявляем указатель на int
        p_age = &age; // адрес переменной age сохраняем в указатель p_age
                      // теперь указатель p_age хранит адрес переменной age
        
        printf("Variable age address: %p\n\n", p_age);

        printf("Enter your age:\n");
        scanf("%d", p_age); // передаём адрес, используя указатель p_age
        
        printf("Age: %d\n", age);
        
        return 0;
}
```

![](./using_pointer_in_scanf.png "Рис. 3 Результ работы програмы Листинг 3")

В этой программе мы передали функции `scanf` адрес переменной не так, как обычно делаем, а через указатель `p_age`, который мы предварительно объявили и присвоили ему значение `&age`. Кстати, допускается сразу инициализировать указатель `p_age` начальным значением: `int *p_age = &age;`.  

Когда мы сохранили адрес переменной `age` в указатель `p_age`, часто говорят, что *"указатель `p_age` указывает на переменную `age`"*. "Указывает" значит "хранит её адрес".

На одну переменную могут указывать сразу несколько указателей, в этом нет ничего страшного. Более того, мы можем присваивать указателю значение, хранящееся в другом указателе, главное чтобы совпадали их типы. Проиллюстрируем это на следующем простом примере.

Листинг 4. 
```c
#include <stdio.h>

int main(void)
{
        int age = 21, current_year = 2026;

        // создаём три указателя на int 
        int *p_age = &age;
        int *p_year = &current_year;
        int *p_another_pointer = 0; // обнуляем указатель

        // выводим адреса, хранящиеся в указателях
        printf("p_age: %p\n", p_age);
        printf("p_year: %p\n", p_year);
        printf("p_another_pointer: %p\n\n", p_another_pointer);

        p_another_pointer = p_year; // сохраняем адрес из указателя p_year
                                    // в указатель p_another_pointer

        p_age = NULL;  // обнуляем указатель через NULL (нулевой указатель)             
        
        printf("p_age: %p\n", p_age);
        printf("p_year: %p\n", p_year);
        printf("p_another_pointer: %p\n\n", p_another_pointer);

        return 0;
}
```

![](./using_null_pointer.png "Рис. 4 Использование нулевого указателя и присваивания указателей")

После объявления, пока мы не сохранили в указателе какое-нибудь значение, в нём, как и в обычной переменной, хранится "мусор". Чтобы "обнулить" указатель можно использовать число `0`, но лучше это делать через специальный =нулевой указатель NULL= -- адрес, который гарантированно не указывает ни на какой объект (обычно адрес, состоящий из нулей).

Разберём строку `p_another_pointer = p_year;`. Т.к. указатель это обычная переменная, то здесь происходит довольно стандартная вещь. Мы берём значение, записанное в переменной `p_year`, и сохраняем это значение в переменную `p_another_pointer`. В итоге получаем, что в обоих переменных записано одинаковое значение -- адрес переменной `current_year`. Что мы и видем на Рисунке 4. Адреса, которые записаны в этих указателях, совпадают. Другими словами, оба указателя указывают на одно и то же место в памяти, на одну переменную `current_year`. 

% **Важно:** При присваивании указателей их типы должны совпадать!

В следующей программе мы попробуем нарушить это требование. Проверьте, что она даже не скомпилируется.

Листинг 5. Ошибки с присваиванием указателей
```c
#include <stdio.h>

int main(void)
{
    int a = 2;
    int b = 10;
    double pi = 3.141592;
    double e = 2.718281;

    int *p_a = &a, *p_b = &b;
    double *p_pi = &pi, *p_e = &e;

    p_a = p_b;  // OK, типы совпадают
    p_e = p_pi; // OK, типы совпадают

    p_b = p_e;  // ERROR, разные типы указателей
    p_pi = p_a; // ERROR, разные типы указателей

    return 0;
}
```

Итак, мы научились создавать указатели и сохранять в них адреса переменных. А теперь самое интересное. Оказывается (ВОТ ТАК НЕОЖИДАННОСТЬ!!1!), используя только адрес переменной, можно изменять данные в памяти, записанные по этому адресу.


## Разыменование указателя

Согласитесь, что название заголовка звучит довольно странно? Мне, честно говоря, слово "разыменование" (англ. dereference) тоже не нравится, оно какое-то непонятное. Думаю, что вы уже и сами догадались, что это такое.

=Разыменование указателя= -- это обращение к ячейке памяти, адрес которой хранится в указателе.

Вместо термина "разыменование" можно использовать на мой взгляд более понятный термин =обращение по адресу=.

Итак, чтобы обратиться к памяти, используя адрес, нужно воспользоваться =оператором обращения по адресу `*`= или =оператором разыменования `*`=. 

Опять звёздочка? Да, и здесь тоже звёздочка. 

Для иллюстрации того, как работать с оператором `*`, давайте рассмотрим следующий пример. 

Листинг 6. Использование оператора обращения по адресу `*`
```c
#include <stdio.h>

int main(void)
{
        int n = 5;     
        printf("n address: %p\n", &n);

        int *p_n = &n;   // теперь p_n указыает на n

        *p_n = 10;  // аналог n = 10;
        printf("n = %d\n", *p_n); // аналог printf("n = %d", n);

        int k = 0;
        k = *p_n;   // аналог k = n; 
        printf("k = %d\n", k);

        *p_n = *p_n * k; // аналог n = n * k
        printf("n = %d\n", *p_n);

        *p_n = 2 * *p_n; // аналог n = 2 * n;
        printf("n = %d\n", *p_n);
        
        return 0;
}
```

Разберём почти построчно код этой программы.

```c
int n = 5;     
printf("n address: %p\n", &n);
```
Сперва создаём переменную `n` и присваем ей значение `5`. Выводим адрес по которому располагается переменная `n`. Для наглядности давайте примерно изобразим текущее состояние памяти.

![](./l5_step0.png "Рис.5 Состояние памяти после создания переменной n")

Начиная с адреса `12b2ffd14`, несколько ячеек памяти, резервируются для хранения значения типа `int`. За этой областью памяти закрепляется имя `n`. После чего в эту область памяти записывается целое число `5`. Конечно, строго говоря, значение `5` будет хранится в памяти в виде последовательности `0` и `1`, но для нашего примера это несущественно, поэтому мы всю область закрасили зелёным цветом и поместили туда значение `5`.


```c
int *p_n = &n;
```

Объявляем переменную-указатель `p_n` (указатель на `int`) и сохраняем в него адрес переменной `n`. Память выглядит следующим образом. 

![](./l5_step1.png)

На рисунке мы видим, что начиная с адреса `12b2ffd08` (я выбрал его случайно), несколько ячеек памяти зарезервированы для хранения адреса. Мы дали этой области памяти имя `p_n` и записали туда адрес переменной `n`. Теперь переменная-указатель `p_n` указывает на переменную `n`, или говоря тоже самое короче: `p_n` указывает на `n`. Для наглядности на рисунке я изобразил этот факт тёмно-синей стрелочкой. Понятно, что в памяти никаких стрелочек нет.

```c
*p_n = 10; 
printf("n = %d\n", *p_n);
```

В первой строке мы используем оператор `*`, чтобы записать по адресу `12b2ffd14` (он хранитсян в указателе `p_n`) значение `10`. Иными словами, мы смотрим, куда указывает стрелочка (куда указывает указатель) и записываем в эту область памяти значение `10`. 

Конечно, мы можем изменить эту же самую область памяти (зелёная область на картинке) и без указателя, используя закреплённое за ней имя `n`.

Итого, теперь у нас есть два альтернативных способа обратиться к этой области памяти: 
- по её имени (используем переменную `n`);
- по её адресу (используем переменную `p_n` и оператор `*`). 

Состояние памяти после выполнения этого кода.

% **Обратите внимание!** 
Запись вида `*имя_указателя` означет, что мы:
1. либо объявляем новый указатель, например: `int *p_n = &n;`.
2. либо обращаемся к тому, на что указывает указатель, например: `*p_n = 10;`. Другими словами: обращаемся к объекту, который находится по адресу, записанному в указателе.

В каком смысле используется символ `*` в строке `printf("n = %d\n", *p_n);`? Очевидно, что во втором, ведь мы здесь явно не создаём новый указатель. Т.е. здесь `*p_n` это то, на что указывает указатель, т.е. переменная `n`. 

```c
int k = 0;
k = *p_n;   // аналог k = n; 
```

Первой строкой мы создаём в памяти переменную с именем `k` (светло-розовая область на картинке) и записываем в неё значение `0`.

Затем во второй строке мы должны сохранить в переменную `k` значение из `*p_n`, т.е. то, на что указывает `p_n`. На что указывает `p_n`? На переменную `n`. Значит в этой строке мы сохраняем в `k` значение из переменной `n` .

Давайте опишем эту же строчку, но теперь на "языке" адресов. 
1. Берём адрес из `p_n` -- `12b2ffd14`. 
2. Смотрим, что расположено в памяти по этому адресу. Там располагается значение `10` (значение переменной `n`). 
3. Берём это значение и записываем его в переменную `k`.

После выполнения этого фрагмента кода, память будет выглядеть следующим образом: 

![](./l5_step3.png)

```c
*p_n = *p_n * k;
```
В этой строке `*p_n` встречается даже два раза, но нас этим не испугаешь!

1. Берём значение, на которое указывает `p_n` (`10` из зелёной области); 
2. Умножаем его на значение переменной `k` (`10` из светло-розовой области); 
3. Полученный результат сохраняем туда, куда указывает указатель `p_n` (по адресу, который хранится в `p_n`, т.е. в зелёную область). 

Теперь в переменной `n` хранится значение `100`. Состояние памяти после указанных манипуляций, представлено на следующем рисунке:

![](./l5_step4.png)

```c
*p_n = 2 * *p_n;
```

1. Берём значение, на которое указывает `p_n` (число `100`);
2. Умножаем его на `2`.
3. Новое значение (`200`) сохраняем по адресу, записанному в указателе `p_n`. 

Мы в очередной раз изменили значение в переменной `n`, через указатель на эту переменную. Итоговое состояние памяти:
![](./l5_step5.png)


Давайте подытожим.

Указатель -- переменная для хранения адресов памяти. 

Указатели предоставляют программисту альтернативный способ работы с памятью -- через адреса. Имея адрес в памяти, мы можем: 
- получить значение, хранящаеся по этому адресу;
- записать по этому адресу новое значение.

Очень многие возможности языка Си реализованы (явно или неявно) через указатели. Например, масса функций стандартной библиотеки языка C, в частности функция `scanf` и любые функции, которые изменяют значение своих аргументов, используют указатели.
