## Дополнительные материалы

**1.** Возможности инструкции `return` ограничены, она может вернуть только одно значение. Если же функция должна вернуть сразу несколько значений, то используются =выходные параметры=. 
Параметр функции, который используется для возврата значений из функции, принято называть =выходным параметром (output parameter)=. 

Используя выходные параметры, функция может вернуть результат своей работы как бы "в обход" инструкции `return`

Для примера напишем функцию `divmod`, которая принимает два целых числа `a` и `b`, а вернуть должна сразу два результата -- частное и остаток от деления `a` на `b`. Вернуть два числа через `return` нельзя, поэтому мы сделаем два выходных параметра `p_quot` и `p_rem`. А `return` будем использовать для того, чтобы сообщить о статусе работы функции `divmod`: будем возвращать `1` -- если всё прошло хорошо, либо `0` -- в случае, когда `b == 0`, т.к. на нуль делить нельзя.

```c
#include <stdio.h>

int divmod(int a, int b, int *p_quot, int *p_rem);

int main(void)
{
        int quot, rem;
        int x, y;
        scanf("%d %d", &x, &y);

        if (divmod(x, y, &quot, &rem)) {
                printf("%d / %d = %d, remainder %d\n", x, y, quot, rem);
        } else {
                printf("Error: division by zero\n");
        }

        return 0;
}


// p_quot и p_rem -- выходные параметры (частное и остаток)
// return: 1 -- успех, 0 -- ошибка (деление на ноль)
int divmod(int a, int b, int *p_quot, int *p_rem)
{
        if (b == 0) {
                return 0; // делить на ноль нельзя, сообщаем об ошибке
        }

        *p_quot = a / b; // частное записываем по адресу p_quot
        *p_rem  = a % b; // остаток записываем по адресу p_rem

        return 1; // всё посчитали успешно
}

```

Мы как бы разделили внутри функции роли:
- `return` отвечает на вопрос "получилось ли посчитать?";
- а конкретные результаты расчётов функция отдаёт через выходные параметры `p_quot` и `p_rem`.


**2.** Есть и другой способ вернуть из функции несколько значений -- использовать =пользовательские структуры данных=. Но в текущей версии Курса мы эту возможность изучать не будем.
