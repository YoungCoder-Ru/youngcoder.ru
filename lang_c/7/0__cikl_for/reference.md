## Дополнительные материалы

**0.** При работе с циклом `for` приходится очень часто к текущему значению переменной прибавлять какое-то число.

Во многих языках программирования, в том числе и в языке Си, можно использовать =сокращённую запись арифметических выражений=.

Например, вместо инструкции `x = x + 10;` можно записать `x += 10;`. Тоже самое можно делать для операторов `-`, `*`, `/`, `%`.

Ещё более часто нам приходится добавлять или отнимать единицу. Для этих операций придумали специальные операторы: =инкремент= и =декремент=.

```c
int x = 0;

x = x + 1;  // увеличиваем текущее значение x на единицу
x += 1;     // сокращённая форма записи того же самого
x++;        // то же самое, но через оператор постфиксного инкремента
++x;        // аналогично, но используем префиксный инкремент

printf("%d\n", x); // выведет 4

x = x - 1;  // уменьшаем текущее значение x на единицу
x -= 1;     // сокращённая форма записи того же самого
x--;        // -1 через оператор постфиксного декремента
--x;        // аналогично, но используем префиксный декремент

printf("%d\n", x); // выведет 0
```

**1.** Чаще всего имя переменной-счётчика делают очень простым, например: `i`, `j`, `k`. С одной стороны -- это дань уважения математическим корням программирования. Но есть и более прагматические цели.
Во-первых, это напоминает нам о "локальности" этой переменной, о том, что её не надо запоминать, т.к. она умрёт сразу, как закончится цикл.

Во-вторых, это снижает когнитивную нагрузку на наш мозг. Видя что-то вроде: `for (int i = 0; i < n; i++)` мы сразу распознаём паттерн: "Ага, тут что-то перебирается по индексу". 

Кроме того, есть негласное соглашение для вложенных циклов: `i` для первого уровня, `j` -- для второго, `k` -- для третьего. 

Конечно, при необходимости, например, если цикл очень длинный и важно не запутаться в контексте, для переменной-счётчика можно и даже нужно использовать осмысленное имя.

**2.** Посмотрите на следующие заголовки циклов:
```c
for (int i = 0; i < 10; i++)

for (int i = 1; i <= 10; i++)
```

Обе эти конструкции предписывают программе выполнить 10 итераций, но первая используется програмистами гораздо чаще. Это связано с тем, что цикл `for` -- это самый главный инструмент при работе с =массивами=. Массивы это специальные переменные, которые хранят не одно, а сразу несколько значений одного типа. А к отдельным значениям можно обратиться по его номеру. Например: `pages[5]`. И вот в массивах как раз нумерация элементов начинается с нуля.  


**3.** До стандарта `С99` объявлять переменные в заголовке цикла `for` было нельзя. И блок инициализация использовался именно для задания начальных значений (инициализации) переменных.


**4.** В цикле `for` можно опустить и управляющее выражение, т.е. записать вот такой бесконечный цикл:
```c
for( ; ; ) {
    // тело цикла
}
```

Этот цикл бесконечный из-за того, что пропущенное управляющее выражение воспринимается как `ИСТИНА`.

Стоит отметить, что и бесконечный цикл иногда может быть достаточно полезен. Например, когда есть множество различных условий по которым нам следует завершить работу цикла. В таких случаях иногда поступают следующим образом: создают бесконечный цикл, а в теле этого цикла, используя условные конструкции `if-else` и инструкции для управления циклами (`continue`, `break`), завершают текущую итерацию или полностью прерывают выполнение цикла. Инструкции управления циклами мы изучим чуть позже в курсе.
